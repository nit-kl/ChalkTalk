# Business Rules - u1-frontend-experience

## Input Rules

1. テーマは必須入力
2. テーマは上限文字数を超えてはならない
3. 無効入力時は生成開始ボタンを無効化するか、押下時に理由を表示する

## YAML Editing Rules

1. YAML編集結果は常に最新内容で検証する
2. 検証結果はユーザーに即時可視化する
3. 構文エラーは行単位で表示する
4. 必須キー不足・型不正・未定義列挙値を区別して表示する

## Preview Rules

1. プレビューは「検証成功済みデータ」のみ対象
2. シーン切替時は対象シーンのみ再描画する
3. アセット未解決時は代替表示し、警告を表示する
4. プレビュー失敗は編集継続を阻害しない

## State Management Rules

1. 生成中は重複生成要求を受け付けない
2. 生成失敗後は再試行可能状態へ戻す
3. ValidationError中でも編集操作は継続可能
4. 画面状態（テーマ、YAML、検証結果、選択シーン）は一貫性を保つ

## Security/Quality Rules (U1適用)

1. エラーメッセージに機密情報を含めない
2. 入力値は表示前に必要なサニタイズ方針を適用する
3. YAML検証ロジックはPBT対象として再利用可能な純粋関数境界を維持する
