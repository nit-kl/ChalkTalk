# ChalkTalk 要件定義書

## 1. 意図分析サマリー

- **ユーザー要求**: 企画書に基づき、新規プロダクトをゼロから作成する
- **要求種別**: New Project（新規開発）
- **スコープ見積**: 複数コンポーネント（フロントエンド、API、アセット管理、動画生成）
- **複雑度見積**: Complex（生成AI連携、クライアント動画生成、段階的機能拡張）
- **前提資料**: `docs/chalktalk_planning.md` と要件確認質問への回答

## 2. プロダクト概要

- **プロダクト名**: ChalkTalk（チョークトーク）
- **コンセプト**: テーマ入力から台本生成、ブラウザ上での動画生成までを一貫して実現する解説動画メーカー
- **価値提案**: インストール不要・低コスト・初心者でも短時間で解説動画を作成可能

## 3. 対象ユーザー

1. YouTube初心者（最優先）
2. 教育系クリエイター
3. 副業・個人開発者
4. 企業の広報・研修担当

## 4. 機能要件（MVP / Phase 1）

### FR-01 テーマ入力とAI台本生成
- ユーザーはテーマ文を入力できること
- システムはAIを利用してYAML形式の台本を生成できること
- YAMLにはシーンID、背景、キャラ情報、セリフ、durationを含むこと

### FR-02 YAML編集機能
- 生成されたYAMLをユーザーが直接編集できること
- 入力内容に対しYAMLバリデーションを行うこと
- バリデーションエラーはユーザーに可視化されること

### FR-03 シーンプレビュー機能
- 編集中のYAMLからシーン単位のプレビューを表示できること
- 背景・立ち絵・黒板文字・字幕の重ね合わせ結果を確認できること

### FR-04 クライアントサイド動画生成
- Canvas APIで背景画像と立ち絵画像をレイヤー合成できること
- 黒板文字と字幕をオーバーレイ描画できること
- MediaRecorder APIでWebM動画を生成できること

### FR-05 動画ダウンロード
- 生成したWebMをブラウザからダウンロードできること

### FR-06 AI台本生成API
- Cloudflare WorkersをAPIプロキシとして利用し、台本生成リクエストを処理できること
- Claude APIとの連携を行えること

### FR-07 アセット管理
- 背景画像・立ち絵をCloudflare R2から配信できること
- アセットマニフェスト（JSON）で論理名と実ファイルパスを解決できること

### FR-08 利用制御
- 台本生成APIに対してレート制限を実装できること

## 5. 非機能要件

### NFR-01 セキュリティ
- セキュリティ拡張ルールを適用する（ユーザー回答: A）
- API入力バリデーション、認可境界、ログ方針、エラーハンドリングを設計段階から定義すること

### NFR-02 品質
- 本番品質を目標とする（ユーザー回答: A）
- 主要機能にはユニットテストと統合テスト方針を定義すること
- 重要ロジックにはプロパティベーステスト（PBT）を適用する（ユーザー回答: A）

### NFR-03 性能
- MVPは短い反復で動作確認可能な応答性を目標とする（ユーザー回答: 反復優先）
- クライアント処理で実用的な動画生成時間を維持すること

### NFR-04 可用性・運用性
- Cloudflare Pages / Workers / R2を前提に、無料枠内で継続運用可能な構成を目指すこと
- 構成変更に備え、代替デプロイ先を選択可能な実装構成を維持すること

### NFR-05 プライバシー
- 最小限データ取得方針を維持し、個人情報DBを持たないこと
- 生成データはユーザー側保存を基本とすること

## 6. 制約・前提条件

- フロントエンド: React + TypeScript + Tailwind CSS
- 状態管理: Zustand
- YAML処理: js-yaml
- 動画生成: Canvas API + MediaRecorder
- ホスティング: Cloudflare Pages
- API: Cloudflare Workers
- ストレージ: Cloudflare R2
- AI台本生成: Claude API（Haiku想定）

## 7. 段階的スコープ

### MVP（今回の主対象）
- FR-01 〜 FR-08

### 将来拡張（Phase 2以降）
- TTS統合（Web Speech API / VOICEVOX）
- MP4変換（FFmpeg.wasm）
- テンプレート機能
- BGM機能
- 字幕出力（SRT/VTT）
- 共有機能、キャラカスタマイズ、独自立ち絵アップロード

## 8. 受け入れ基準（MVP）

1. テーマ入力からYAML台本が生成される
2. YAML編集時にバリデーションが動作する
3. シーンプレビューで背景・立ち絵・文字描画が確認できる
4. WebM動画をブラウザで生成し、ダウンロードできる
5. 台本生成APIにレート制限が適用される
6. アセットはR2マニフェスト経由で解決される

## 9. リスク起点の設計考慮

- MediaRecorderのブラウザ互換差異に対するフォールバック案を用意する
- AIコスト急増対策として、レート制限・キャッシュ・将来BYOKを計画する
- 画像アセット一貫性の担保手順を運用フローに組み込む
- 不適切コンテンツ生成リスクに対し、利用規約と入力制御を整備する
